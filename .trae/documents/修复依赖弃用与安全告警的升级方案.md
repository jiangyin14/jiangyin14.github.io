## 问题概览
- 直接依赖：`eslint@8.57.1`（已 EOL）。
- 传递依赖（锁文件中）：`uuid@3.3.2`、`path-match@1.2.4`、`glob@7.2.3`、`rimraf@3.0.2`、`mkdirp@0.*`、`source-map@0.8.0-beta.0`、`@humanwhocodes/*`、`inflight`、`gauge`、`are-we-there-yet`、`npmlog`。
- 溯源：多数问题来自 `vercel` CLI 生态（`@vercel/nft` 等）和 PWA 构建链（`workbox-build`）。当前代码库未直接使用这些旧包。

## 升级策略
- 升级为主、替换为辅，尽量通过上游包的版本提升来清理传递依赖。
- 保持 Node≥20 与现有 VitePress/Vue3 工程兼容。

## 具体改动
- ESLint：迁移到 `eslint@^9` 与 Flat Config，移除 `eslint-config-airbnb-base`（其对 v9 支持不稳）。
  - 新增：`@eslint/js`；保留：`eslint-plugin-vue`、`eslint-plugin-import`。
  - 把 `.eslintrc.js` 改为 `eslint.config.js`，示例：
    ```js
    import js from '@eslint/js'
    import vue from 'eslint-plugin-vue'
    export default [
      js.configs.recommended,
      vue.configs['flat/recommended'],
      {
        languageOptions: { ecmaVersion: 'latest', sourceType: 'module' },
        rules: {
          quotes: ['warn', 'double'],
          'import/extensions': 'off',
          'no-console': 'off',
        },
      },
    ]
    ```
- Vercel CLI：升级 `vercel@latest`，以替换其旧的传递依赖链（`uuid@3`、`path-match@1`、`glob@7`、`gauge`、`npmlog` 等）。
- PWA 集成：升级 `@vite-pwa/vitepress@latest`，提升到较新的 Workbox（7.1+/7.3），避免 `source-map@0.8.0-beta.0` 与旧 `glob@7`。
- 其他：不在代码中直接使用 `rimraf`、`mkdirp`，但若后续出现直接依赖，统一切到 `rimraf@^5`、`mkdirp@^1`（Promise API）。

## 变更命令（pnpm）
- 升级 ESLint：
  - `pnpm up -D eslint@^9 @eslint/js eslint-plugin-vue@^9 eslint-plugin-import@^2.31`
  - `pnpm remove eslint-config-airbnb-base`
- 升级工具链：
  - `pnpm up -D vercel@latest`
  - `pnpm up @vite-pwa/vitepress@latest`
- 安装与刷新锁：
  - `pnpm install`

## 兼容性与注意事项
- ESLint v9 只支持 Flat Config，Airbnb Base 需移除或改为同等规则集合。
- Workbox 升级可能微调 PWA 选项（如 `dontCacheBustURLsMatching`），现有配置基本可复用。
- `uuid@3` 曾在特定情况下使用 `Math.random()`，升级链可规避该风险；`debug` 版本需 ≥4.3.1（当前锁中已是 4.4.x）。

## 验证步骤
- `pnpm lint` 无报错（规则与插件加载正常）。
- `pnpm dev` 启动后观察安装日志：弃用与安全告警显著减少/消失。
- 如仍有个别传递包未清理，记录来源并按“升级上游/替换等价包”进一步处理。

## 预期效果
- 消除 `eslint@8` EOL 告警；
- 大幅减少旧链路包（`uuid@3`、`glob@7`、`path-match@1` 等）产生的弃用与风险提示；
- 保持现有功能与开发体验不变。
